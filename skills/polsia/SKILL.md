# Polsia

Use this as the master front door for the Polsia workflow.

## When to use

Use this skill when the user asks to:

- validate a startup idea before building
- plan an MVP or execution path
- ship with AI/no-code tooling
- improve positioning, brand, or launch copy
- run continuous execution loops like Polsia’s “God Mode”
- consolidate Polsia phase skills behind one entrypoint

## Read first

If you need source detail, read `references/upstream-notes.md`. It maps the live sitemap-driven corpus and the blog themes this skill is based on.

## Subcommands

- `/polsia start` — define the problem, audience, wedge, and proof you need before code.
- `/polsia plan` — reduce the validated idea into a small, testable plan and MVP scope.
- `/polsia build` — choose the lightest path to a working product, keeping context and ownership intact.
- `/polsia brand` — tighten the message, tone, and proof so the offer feels credible.
- `/polsia launch` — bundle product, distribution, and launch motion into one release plan.
- `/polsia scale` — keep the system running after launch; focus on ops, support, and growth loops.

## Operating rules

- Prefer evidence over enthusiasm.
- Prefer one coherent workflow over disconnected tools.
- Keep the next action concrete: decision, artifact, or test.
- If the request is broad, ask which subcommand they mean.
- If live Polsia pages change, refresh the upstream notes before rewriting the skill.
